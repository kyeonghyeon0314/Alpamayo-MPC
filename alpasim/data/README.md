# Alpasim Data
This directory contains all subdirectories which we will need to mount to the various docker containers to run alpasim.

1. `nre-artifacts`
    * Contains the `.usdz` files which are scene reconstructions for neural rendering with NRE
    * Also used by the runtime to obtain ground truth vehicle trajectories and physics for access to scene meshes
2. `trafficsim` _(coming soon)_
    * Will contain scene road maps and configuration for the traffic simulator
3. `config`
    * Contains the configuration file for the runtime, split in two parts
        * `network-config.yaml` which specifies the addresses of the (potentially multiple) replicas of each microservices such that the runtime is aware of available resources. This should be in sync with the content of of `docker-compose/docker-compose.yaml` (ideally autogenerated based on it)
        * `user-config.yaml` which specifies the scenes to simulate, duration, etc
4. `rollouts`
    * The output directory for logs produced by the runtime in `.asl` format (size-delimited protobuf files)
5. `drivers`
    * Local driver assets (e.g. AR1 checkpoints). The wizard creates this dir if missing when using `+deploy=local`.

**HuggingFace models (e.g. AR1):** Set the `HF_HOME` environment variable to control where models are cached (defaults to `~/.cache/huggingface`). Download models with `huggingface-cli download nvidia/Alpamayo-R1-10B` (you may need `huggingface-cli login` first). The wizard mounts the HF cache into the driver container.

The `auto-init.sh` and `auto-init-aws.sh` scripts automatically download example files necessary in each subdirectory. `auto-init.sh` prompts you to choose which model(s) to download (VaVAM-B and/or AR1).
